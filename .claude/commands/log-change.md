Помогаю добавить запись в CHANGELOG.md проекта.

## Контекст

Проект следует формату [Keep a Changelog](https://keepachangelog.com/en/1.0.0/) и [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## Текущий CHANGELOG

@docs/CHANGELOG.md

## Процесс добавления записи

1. **Определить тип изменения**:
   - **Added** - новая функциональность
   - **Changed** - изменения в существующей функциональности
   - **Deprecated** - функции, которые скоро будут удалены
   - **Removed** - удаленная функциональность
   - **Fixed** - исправления багов
   - **Security** - изменения связанные с безопасностью

2. **Спросить описание изменения** (краткое, но информативное)

3. **Добавить запись** в секцию `[Unreleased]` под соответствующую категорию
   - Если категории еще нет в `[Unreleased]`, создать ее
   - Сохранить порядок категорий: Added, Changed, Deprecated, Removed, Fixed, Security

## Формат записи

Каждая запись должна быть:
- Лаконичным bullet point
- Название компонента выделено жирным: `**Название**`
- Дополнительные детали после названия
- Использовать настоящее время

Примеры:
```markdown
### Added
- **Database Browser** - визуализация таблиц и данных с фильтрацией и сортировкой
- **Query Editor** - интерактивный редактор SQL запросов с подсветкой синтаксиса

### Fixed
- **Visual Alembic UI** - исправлена ошибка при откате первой миграции
```

## Важные правила

- ✅ Всегда добавлять в секцию `[Unreleased]` (не в версионированный релиз)
- ✅ Соблюдать порядок категорий
- ✅ Быть кратким но информативным
- ✅ Использовать русский язык для описаний
