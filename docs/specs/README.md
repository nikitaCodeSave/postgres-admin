# Specifications - Реализованные фичи

Этот каталог содержит детальные спецификации **реализованных** фич проекта.

## Цель

После того как фича из `/docs/backlog/features` реализована, мы создаем здесь полную спецификацию с описанием:
- Как работает фича
- Какие API endpoints реализованы
- Какие компоненты созданы
- Примеры использования
- Известные ограничения

## Формат файлов

`NNN-feature-name.md` где:
- `NNN` - порядковый номер (001, 002, 003...)
- `feature-name` - краткое имя фичи

## Отличие от ADR

- **ADR** (Architecture Decision Records) - **почему** мы приняли то или иное архитектурное решение
- **Specs** - **что** и **как** работает в реализованной фиче

## Workflow

1. Фича реализована и закоммичена
2. Создаем спецификацию в `/docs/specs/NNN-feature-name.md`
3. Обновляем CHANGELOG.md
4. Файл из `/docs/backlog/features` можно удалить или пометить как Completed

## Шаблон

См. `spec-template.md` для структуры спецификации.
